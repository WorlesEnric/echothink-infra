# Outline — EchoThink 的团队知识库

## 服务概述

Outline 是 EchoThink 基础设施中的协作式知识库平台，当前部署版本为 1.5.0。在游戏开发团队的日常工作中，知识的积累与流转往往决定了项目的推进效率。游戏设计文档、世界观设定集、角色美术风格指南、技术架构规范、引擎使用手册——这些文档散落在不同成员的个人设备中或者淹没在即时通讯工具的消息流里，不仅难以检索，更容易在版本迭代中产生混乱。Outline 正是为解决这些痛点而引入的：它提供了一个结构清晰、支持实时协作编辑的团队 Wiki，让所有项目知识都能以文档集合的形式组织起来，并通过强大的全文搜索能力让任何团队成员（包括即将上线的 AI 智能体）都能快速找到所需信息。

Outline 的文档编辑器基于 Markdown 语法，对技术团队非常友好，同时也提供了富文本编辑的可视化操作方式，让非技术背景的游戏策划和美术设计师同样能够轻松上手。文档支持嵌套结构，可以按照项目、模块、功能等维度建立多层级的文档集合，这与游戏项目中常见的内容组织方式天然契合：一个大型游戏项目可以按照"世界观设定"、"关卡设计"、"角色系统"、"战斗机制"、"UI/UX 规范"等类别建立顶层集合，每个集合内再按照迭代版本或功能模块进行细分。实时协作编辑功能允许多位团队成员同时修改同一篇文档，所有变更即时同步，彻底消除了"谁的版本是最新的"这类协作困境。

## 部署架构与配置

Outline 在 EchoThink 中以 Docker 容器方式部署，使用官方镜像 `outlinewiki/outline:1.5.0`。容器内部应用监听 3000 端口，映射到宿主机的 `127.0.0.1:3200`，所有外部流量通过 Nginx 反向代理转发。Nginx 配置中定义了 `outline-backend` 上游服务器指向容器的 3000 端口，并配置了 WebSocket 协议升级支持，这是实时协作编辑功能所必需的——当多位用户同时编辑一篇文档时，客户端与服务器之间通过 WebSocket 连接保持文档状态的实时同步。`client_max_body_size` 设为 50MB，足以支持大多数文档附件的上传需求。

Outline 的数据持久化层完全依托 EchoThink 的共享基础设施。工作流数据和文档内容存储在共享 PostgreSQL 16 实例中的 `outline` 数据库，通过连接字符串 `postgres://outline_user:${OUTLINE_DB_PASSWORD}@postgres:5432/outline` 进行访问，数据库连接池配置为最小 2 个连接、最大 10 个连接，在保障响应性能的同时避免对共享数据库实例造成过大压力。Redis 实例用于缓存和会话管理，连接至共享的 Redis 服务。文档中的附件和图片则通过 S3 兼容协议存储在 MinIO 对象存储中，使用名为 `outline-data` 的专用存储桶。这种架构将计算与存储分离，使得 Outline 容器本身是无状态的，便于水平扩展和故障恢复。容器启动时会依次等待 PostgreSQL、Redis 和 MinIO 三个依赖服务通过健康检查，确保所有存储后端就绪后才开始服务初始化。

关键环境变量与端口信息如下：

- 内部端口：3000
- 宿主机映射：127.0.0.1:3200
- 外部访问：https://outline.${DOMAIN}（经 Nginx 反向代理）
- `OUTLINE_SECRET_KEY`：应用加密密钥
- `OUTLINE_UTILS_SECRET`：工具函数加密密钥
- `OUTLINE_DB_PASSWORD`：PostgreSQL 数据库连接密码
- `FILE_STORAGE_UPLOAD_MAX_SIZE`：附件上传大小限制（默认 25MB）

## 统一身份认证与安全

Outline 通过 OIDC（OpenID Connect）协议与 EchoThink 的统一身份认证服务 Authentik 实现了单点登录集成。当团队成员访问 Outline 时，系统会自动跳转到 Authentik 的登录页面完成身份验证，验证通过后携带授权令牌返回 Outline。这种集成方式意味着团队成员无需为 Outline 单独维护账号密码，使用统一的 EchoThink 账号即可访问所有平台服务。OIDC 配置中定义了完整的授权流程端点，包括授权 URI、令牌交换 URI、用户信息 URI 和登出 URI，所有认证流量都通过 HTTPS 加密传输。请求的权限范围（scope）包括 `openid`、`profile` 和 `email`，确保 Outline 能够获取用户的基本身份信息用于账号自动创建和权限分配。

在安全防护方面，Outline 启用了速率限制器，配置为每 60 秒内最多允许 1000 次请求，有效防止恶意爬取或暴力攻击。SMTP 邮件服务已配置完毕，支持文档评论通知、共享邀请和密码重置等场景。所有敏感配置均通过环境变量注入，不存在硬编码的凭证信息。

## 文档管理与全文搜索

Outline 提供了完善的 RESTful API，支持以编程方式进行文档的创建、读取、更新、删除以及搜索操作。这个 API 接口是 Outline 在 EchoThink 生态中发挥更大价值的关键——它不仅允许 n8n 的自动化工作流对知识库进行批量操作，更为即将上线的 Claw Cluster AI 智能体提供了读写知识库的程序化通道。API 支持按集合、标签和关键词进行文档检索，返回结构化的 JSON 数据，便于下游系统解析和处理。

全文搜索是 Outline 的核心能力之一。基于 PostgreSQL 的全文索引技术，Outline 能够在毫秒级时间内搜索所有文档的标题和正文内容，支持关键词高亮和相关度排序。对于游戏开发团队来说，这意味着当程序员需要查找某个游戏机制的设计规格时，只需在搜索框输入关键词，就能从可能包含数百篇文档的知识库中精准定位到目标文档。文档的版本历史功能记录了每次编辑的变更详情，团队可以随时回溯到任意历史版本，这在游戏设计方案频繁迭代的过程中尤为重要。

## 游戏开发场景中的应用价值

在游戏开发团队的实际使用中，Outline 知识库的价值体现在项目知识的结构化沉淀和高效流转两个维度。从知识沉淀的角度，团队可以在 Outline 中建立完整的游戏设计知识体系：世界观设定文档描述游戏的背景故事、文化体系和地理结构；关卡设计文档详细记录每个关卡的目标、难度曲线和谜题逻辑；美术风格指南定义角色的视觉规范、场景的色彩方案和 UI 的交互原则；技术规范文档则涵盖引擎架构说明、编码标准、API 接口文档和性能优化指南。这些文档不是静态的存档，而是随着项目推进持续更新的活文档，所有团队成员都可以在权限范围内进行编辑和评论。

从知识流转的角度，Outline 的实时协作能力和 API 接口打通了人与人、人与工具、工具与工具之间的信息壁垒。游戏策划在 Outline 中更新了战斗系统的数值设计后，程序员可以立即在同一文档中看到变更内容并添加技术可行性的评论。n8n 自动化工作流可以在每日构建完成后自动将构建报告写入 Outline 的"每日报告"集合。当新成员加入团队时，一套组织良好的 Outline 知识库就是最好的上手指南，大幅缩短了新人的学习曲线。

## 与 Claw Cluster 的协同展望

Claw Cluster（AI 员工集群）作为 EchoThink 平台即将上线的智能核心，将与 Outline 知识库建立深度的双向交互关系。在信息获取方向，Claw Cluster 中的 AI 智能体将通过 Outline 的 API 读取游戏设计文档、技术规范和项目上下文信息，这些文档内容将成为 AI 智能体理解项目需求和执行任务时的关键上下文。例如，当一个 AI 智能体被分配编写某个游戏系统的代码时，它会首先从 Outline 中检索该系统的设计文档和技术规范，确保生成的代码与团队约定的设计方案和编码标准保持一致。

在信息输出方向，AI 智能体完成任务后会将成果回写到 Outline 知识库中。代码审查 AI 生成的审查报告、测试 AI 生成的测试覆盖率分析、策划辅助 AI 生成的关卡设计方案——这些 AI 产出的内容都将以结构化文档的形式存入 Outline 相应的集合中，供人类团队成员审阅、修改和批准。Outline 由此成为了人类团队成员与 AI 智能体之间的知识交换枢纽：人类在 Outline 中表达意图和需求，AI 在 Outline 中呈现分析和产出，双方围绕知识库中的文档展开异步协作。这种以知识库为中心的人机协作模式，让团队的每一份知识积累都能被所有参与者——无论是人类还是 AI——充分利用，最大化了知识资产的投资回报。
